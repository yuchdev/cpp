Советы и резюме:
1. Контейнер, подходящий для большинства задач - вектор.
2. Всегда оценивайте сложность О() необходимой операции.
3. Сортировать и искать можно с множеством критериев.
4. При вставке-удалении предпочитайте работу с концом контейнера
   работе с его началом, в целях переносимости.
5. Если элементы должны быть упорядочены, выбирайте map (не hash_map!)
6. Когда важна скорость поиска, выбирайте hash_map
7. Когда для элементов нельзя определить операцию "меньше", выбирайте hash_map
8. Для проверки нахождения ключа в map используйте find()
9. Когда ключ является единственным значением, пользуйтесь set
10.Пользуйтесь equial_range для поиска диапазона элементов в multimap/multeset
11.Пользуйтесь bitset для битовых операций, преобразования бит в строку.
12.Определяйте operator() вместо указателей на функции
13.Используйте встроенные методы списка вместо соответствующих алгоритмов (например list::sort, map::find)
14.Сложность операции list::splice (удалить и вставить) зависит от сложности list::size
15.При вставке в map можно указать элемент, после которого производится  (некоторая оптимизация)
16.С функцией splice надо быть осторожнее при передаче итераторов - они должны указывать на один список и
   быть валидны.
17.Нельзя сливать неотсортированные списки с помощью list::merge.
18.Функции поиска/удаления/уникальности списка принимают предикаты.
19.deque объединяет свойства вектора и списка
20.Адаптеры последовательностей: stack и queue
21.Адаптер priority_queue может принимать предикат сравнения (сам контейнер основан на куче)
22.Удобно определять typedef для объявлений map и прочих многословных типов.
23.Ассоциативный массив требует, чтобы для ключа существовала операция "меньше".
24.При вставке в map возвращается pair<>, содержащий итератор на вставленную строку
   и логический флаг bool, означающий успешную или неуспешную вставку.
25.Итерация по map производится по порядку сортировки.
26.Когда ключ по операции [] не находится, создается значение по умолчанию
27.Элементы из map можно удалять по одному или диапазонами.
28.Большинство контейнеров дают сильную гарантию исключений, кроме
   ассоциативных на вставке диапазонов, сортировки списков.
   Копирование, swap, предикаты элементов не должны кидать исключений
* Можно попробовать выбрать лучшую хеш-функцию (CRC32, SHA, FastHash, XOR)
